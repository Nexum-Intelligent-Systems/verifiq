# ADR-001 · Canonical review-dispatch stack & generated-file policy

**Status:** Accepted · **Date:** 2026-06-09 · **Owner:** `main` integrator

## Context

VerifIQ has been built by two parallel Claude Code sessions on separate
branches. They independently implemented the same requirement (the file-20 §2
job queue) two different ways, and the implementations repeatedly collided on
merge — leaving `main` red, or (worse) green-locally-but-broken-from-clean. Two
artefacts of that collision keep recurring:

- **Two review-dispatch implementations.**
  - **Queue runner (CANONICAL):** `src/convex/jobs.ts` (`internalMutation`/
    `internalQuery` job table), `src/convex/runner.ts` (`tick` `"use node"`
    action), `src/convex/crons.ts` (1-minute drain), over
    `src/orchestrator/runner.ts` (`drainQueue`), `council.ts` (`createCouncil`)
    and `convex-persistence.ts` (`ConvexPersistence`).
  - **Single-action dispatch (RETIRED):** `src/convex/review.ts` (`runReview`),
    `src/convex/reviewData.ts` (`requestReview` / `resumeStalled`, 15-min cron),
    a `review_inputs` table, `src/orchestrator/convex-port.ts`,
    `src/llm/cache-convex.ts`, `bundledPromptLoader()`.
- **Two prompt-bundle artefacts / a gitignored-but-imported file.** The code
  imports `PROMPT_BUNDLE` from a gitignored `src/agents/prompts.bundle.ts`; an
  earlier track committed a different `prompts.generated.ts` (`BUNDLED_PROMPTS`)
  that the build script wrote instead — so a clean checkout couldn't build.

## Decision

1. **The queue runner over `internal*` Convex functions is the one canonical
   review stack.** All `jobs`/queue functions are `internal*`, driven only by
   the cron-scheduled `tick`. The single-action `runReview` / `requestReview`
   dispatch is **retired** — do not import, extend, or re-add it. The authed
   public entry point (post-Clerk) wraps `enqueueJob` behind project membership;
   it does **not** reintroduce `review.ts`.

2. **One canonical directory per concern.**
   - Classifier: **`src/classifier/`** (imported by `tests/classifier.test.ts`
     and `src/extraction/pdf.ts`). `src/classify/` is retired — do not recreate.
   - Inference cache: **`src/llm/cache.ts`** + **`src/convex/cache.ts`**.
     `src/llm/cache-convex.ts` is retired.
   - Persistence port: **`src/orchestrator/convex-persistence.ts`**.
     `src/orchestrator/convex-port.ts` is retired.

3. **One generated prompt bundle, gitignored, regenerated by the script.**
   `scripts/bundle-prompts.mjs` writes **`src/agents/prompts.bundle.ts`**
   exporting **`PROMPT_BUNDLE`**, gitignored. No second bundle artefact, and no
   committed generated file. Generated files (this bundle, Convex `_generated/`)
   are **gitignored + regenerated in CI**, never committed.

## Enforcement

- `scripts/check-hygiene.mjs` (CI step `check:hygiene` + `tests/hygiene.test.ts`)
  fails on duplicate `package.json` keys, any tracked-but-gitignored file, and
  any committed `*.generated.ts` / `*.bundle.ts`.
- CI (`.github/workflows/ci.yml`) builds from a **clean checkout** — it produces
  every generated artefact itself, so a missing/mis-named generated file fails
  CI even when a dirty local tree hides it.

## Consequences

- Future sessions converge on the queue runner instead of re-adding the
  alternative. Any PR that adds a second dispatch path, a second classifier
  directory, or a second prompt-bundle artefact is rejected by the guard + review.
- Removing the retired track is a one-time cleanup; the guard prevents re-addition.
